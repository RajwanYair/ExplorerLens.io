# DarkThumbs v5.3.0 - Release Notes

**Release Date:** January 8, 2026  
**Codename:** "Engine Integration"  
**Status:** ✅ Production Ready

---

## 🎯 Overview

Version 5.3.0 represents a **major architectural milestone** - the successful integration of the new DarkThumbs Engine library with the legacy CBXShell COM extension. This hybrid approach enables modern thumbnail processing while maintaining backward compatibility.

---

## ✨ What's New

### Architecture Transformation

- **Engine Library**: Standalone `DarkThumbsEngine.lib` (1.93 MB)
  - Clean C++ API with zero COM dependencies
  - Modular decoder architecture
  - Pipeline-based thumbnail processing
  - Comprehensive unit test coverage (22/22 tests passing)

- **EngineAdapter Bridge**: Seamless COM ↔ Engine integration
  - Automatic fallback to legacy code if Engine fails
  - Configurable Engine initialization
  - Statistics and performance monitoring
  - Detailed logging for troubleshooting

### Active Decoders (v5.3.0)

| Decoder | Formats Supported | Status |
|---------|------------------|--------|
| **ImageDecoder** | JPEG, PNG, BMP, GIF, TIFF, ICO | ✅ Active |
| **WebPDecoder** | WebP (static & animated) | ✅ Active |
| **AVIFDecoder** | AVIF (AV1 images) | ✅ Active |
| **ArchiveDecoder** | ZIP, CBZ, RAR, CBR, 7z, CB7 | ✅ Active |

### Legacy Decoders (Still in CBXShell)

These remain in CBXShell for now (will be migrated to Engine in v6.0):
- JXL (JPEG XL)
- HEIF/HEIC
- PDF documents
- Video formats (MP4, MKV, AVI, WebM)
- Audio formats (MP3, FLAC, WAV)
- Fonts (TTF, OTF, WOFF)
- RAW photos (CR2, NEF, ARW, DNG)

## 🏗️ Technical Details

### Build Configuration

```
Engine:   Release x64, /MD runtime, C++20
CBXShell: Release x64, /MD runtime, C++20
Output:   CBXShell\x64\Release\CBXShell.dll (1.32 MB)
```

### Engine Integration Points

1. **CCBXShell::FinalConstruct()**
   - Creates EngineAdapter instance
   - Initializes Engine with default configuration
   - Registers all 4 decoders
   - Sets `m_useEngine = true` on success

2. **CCBXShell::GetThumbnail()**
   - Routes requests to Engine first
   - Falls back to legacy code if Engine fails
   - Returns HBITMAP with alpha channel

3. **CCBXShell::FinalRelease()**
   - Cleanly shuts down Engine
   - Releases all resources

## 🚀 Installation

### Prerequisites
- Windows 10/11 (64-bit)
- Visual C++ 2015-2026 Redistributable (x64)
- Administrator rights (for registration)

### Registration

```powershell
# Unregister old version (if installed)
regsvr32 /u "C:\...\CBXShell.dll"

# Register new version
regsvr32 "C:\Users\ryair\...\CBXShell\x64\Release\CBXShell.dll"
```

### Verification

1. Right-click any image file in Explorer
2. Thumbnails should appear immediately
3. Check Windows Event Viewer → Application
4. Look for "DarkThumbs" events with Engine logging

## 📝 Migration Notes for Developers

### For CBXShell Maintainers

The Engine is now the **primary** thumbnail provider. Legacy code remains as a **safety net only**.

To add new format support:
1. Create decoder in `Engine/Decoders/` (inherit from `IThumbnailDecoder`)
2. Register in `EngineAdapter::RegisterDecoders()`
3. Add unit tests in `Engine/Tests/`
4. Legacy fallback handles edge cases automatically
