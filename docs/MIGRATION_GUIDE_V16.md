# ExplorerLens v15 → v16 Migration Guide

**From:** v15.x "Zenith" series  
**To:** v16.0.0 "Horizon"  
**Breaking changes:** None (v16.0.0 is ABI-compatible with v15.x)

---

## Overview

v16.0.0 "Horizon" is a backwards-compatible major version bump. All existing
plugin binaries compiled against the v15.x Plugin SDK continue to work without
recompilation. The only changes that require attention are:

1. New `AnonymousTelemetry` consent registration (opt-in only)
2. WinUI 3 `LENSManager.WinUI.exe` replaces the WTL `LENSManager.exe` as the
   default GUI (WTL binary still ships for headless/Server environments)
3. i18n string resources now loaded from `Engine/i18n/strings_<lang>.rc`

---

## For Plugin Developers

### SDK ABI Compatibility

The Plugin SDK C ABI (`SDK/plugin_api.h`) is unchanged between v15.x and v16.0.0.
Plugins compiled with v15.x SDK headers will load, pass trust verification,
and function correctly under v16.0.0.

```cpp
// No changes required in plugin_api.h structs
typedef struct LENS_PluginInfo_v1 {
    uint32_t sdk_version;     // Report LENS_SDK_VERSION (still 1)
    ...
} LENS_PluginInfo;
```

### New Optional Capabilities

Plugin authors may now implement the following optional interfaces
(returning `E_NOTIMPL` from v15.x plugins is fine):

| Method | Purpose |
|--------|---------|
| `ILENSPluginV2::GetA11yDescription()` | WCAG accessible name for plugin-decoded formats |
| `ILENSPluginV2::GetTelemetryTag()` | Plugin-specific telemetry key (prefix: `plugin.`) |

## For System Administrators

### New Registry Keys (v16.0.0)

| Key | Type | Default | Purpose |
|-----|------|---------|---------|
| `HKCU\Software\ExplorerLens\Telemetry\Enabled` | DWORD | 0 | Opt-in anonymous telemetry |
| `HKCU\Software\ExplorerLens\UI\PreferWinUI` | DWORD | 1 | Use WinUI 3 manager (0 = WTL) |
| `HKLM\Software\ExplorerLens\i18n\Language` | SZ | `auto` | Force UI language (en/fr/de) |

### Group Policy (ADMX)

New ADMX template shipping with v16.0.0: `ExplorerLens.admx`

- `Computer\ExplorerLens\DisableTelemetry` — Machine-level telemetry override
- `Computer\ExplorerLens\AllowedGPUBackend` — Restrict backend (D3D12/D3D11/CPU)
- `User\ExplorerLens\CacheBudgetMB` — Per-user cache budget cap

## For End Users

### Installer Upgrade Path

The v16.0.0 MSI installer (`ExplorerLens-16.0.0-x64.msi`) performs an in-place
upgrade from any v15.x installation.  No manual uninstall required.

Upgrade notes:
- COM CLSID unchanged: `{9E6ECB90-5A61-42BD-B851-D3297D9C7F39}`
- Cache directory preserved: `%LOCALAPPDATA%\ExplorerLens\cache`
- Settings preserved via HKCU registry migration

### New Default Launcher

`LENSManager.WinUI.exe` is now the default manager executable.
The legacy `LENSManager.exe` (WTL) remains in the installation directory
and continues to work on Windows Server and headless environments.
