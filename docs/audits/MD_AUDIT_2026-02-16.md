# Markdown Audit — 2026-02-16

## Scope
- Root markdown documents
- `docs/` markdown set
- script READMEs and planning/status references

## Summary
This audit identifies roadmap/status drift, stale links, and duplicated guidance. Items below were either fixed in this cleanup pass or explicitly moved into `MASTER_PLAN.md` backlog.

## Findings

### 1) Version and status drift
- Mixed version narratives across docs (`v5.x`, `v6.0`, `v6.2`) caused contradictory project state.
- Several docs referenced sprint phases as current despite being historical snapshots.

### 2) Broken/stale internal links
- References to deleted or absent paths were present, including:
  - `docs/build/*`
  - `docs/planning/*`
  - `release-scripts/*`
  - legacy script path variants

### 3) Feature status contradictions
- `README.md` and `KNOWN_ISSUES.md` disagreed on implementation status for modern formats (notably JXL/HEIF).
- Planned/future wording persisted in user-facing docs for already integrated capabilities.

### 4) Script documentation mismatch
- Some script docs and wrappers referenced files/locations not present in current repository layout.
- Structure verification script expected directories/files that no longer exist after consolidation.

### 5) Roadmap authority spread
- Multiple docs included sprint/roadmap claims in conflict with intended single-source planning.

## Actions taken in this pass
1. Rebuilt `MASTER_PLAN.md` as unified detailed roadmap with explicit backlog from this audit.
2. Updated `docs/INDEX.md` to remove stale links and align quick references with existing docs.
3. Updated root `README.md` status/version consistency.
4. Updated `KNOWN_ISSUES.md` outdated planned item status for modern format support.
5. Repaired stale script path usage in utility/build scripts.
6. Repaired stale checks in `scripts/verify-project-structure.ps1`.
7. Updated duplicate-cleanup script canonical path guidance.

## Remaining recommended follow-ups
- Add automated docs link checker in CI (internal markdown links and known canonical files).
- Add a simple “version authority” policy: only `CHANGELOG.md` and `MASTER_PLAN.md` define active release state.
- Convert historical sprint docs to archived status-only records to avoid accidental use as current plan.

## Result
- Planning authority consolidated.
- Key stale links and script-path drift corrected.
- Backlog items promoted into the main execution plan.
