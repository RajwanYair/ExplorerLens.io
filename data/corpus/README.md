# ExplorerLens Test Corpus

This directory holds **decoder validation test files** for ExplorerLens.

## Structure

```text
data/corpus/
├── MANIFEST.json       — Master index: format → decoder → expected performance
├── images/             — Raster images (JPEG, PNG, WebP, AVIF, HEIC, JXL, …)
├── raw/                — Camera RAW files (DNG, CR2, NEF, ARW)
├── documents/          — Documents (PDF, EPUB, XPS)
├── archives/           — Archive files (CBZ, CBR, ZIP, 7z, tar.gz)
├── models/             — 3D models (glTF, GLB, OBJ, STL)
├── fonts/              — Font files (TTF, OTF, WOFF2)
├── video/              — Video clips (MP4, MKV, WebM)
└── audio/              — Audio files (MP3, FLAC)
```

## Generating the Corpus

```powershell
# Generate synthetic test files for all formats
.\tools\Generate-Corpus.ps1

# Validate decoders against corpus (requires built EngineTests.exe)
.\tools\Validate-Corpus.ps1

# Optional: download free-licensed real-world samples
.\tools\Generate-Corpus.ps1 -DownloadSamples
```

## File Size Policy

- Synthetic files generated by `Generate-Corpus.ps1` are **small by design** (< 100 KB each)
- Real-world samples must be < 10 MB each
- Files > 10 MB must NOT be committed — add your own real samples locally
- The `.gitignore` in this directory excludes large files automatically

## CI Validation

The workflow `.github/workflows/corpus-validation.yml` runs `Validate-Corpus.ps1`
against every PR that touches decoder code. See MANIFEST.json for per-format latency baselines.

## Contributing Samples

1. Ensure the sample file is freely licensed (CC0 / public domain)
1. Add it to the appropriate subdirectory
1. Update `MANIFEST.json` with the `test_file` field
1. Run `Validate-Corpus.ps1` locally to confirm it decodes correctly
1. Open a PR — the corpus-validation CI will verify it
